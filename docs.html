Getting Started with ProofFix

Everything you need to install, configure, and use ProofFix for safe code evolution.

Installation

ProofFix can be installed via npm, Homebrew, or as a standalone binary.

npm (Recommended)

npm install -g prooffix

Homebrew (macOS/Linux)

brew tap prooffix/tap
brew install prooffix

Binary Download

Download the latest release from GitHub Releases for your platform.

Verify Installation

prooffix --version
# ProofFix v1.0.0

Quick Start

Get started in under 2 minutes:

1. Navigate to your project

cd /path/to/your/project

2. Run a scan

prooffix scan .

This analyzes your codebase and reports any issues found.

3. Auto-fix issues

prooffix scan . --auto-fix

Automatically apply verified fixes. A Git snapshot is created before any changes.

4. Review results

ProofFix generates a clear report showing what was fixed and what needs manual attention.

Command Reference

prooffix scan [path]

Scan a directory or file for issues.

# Scan current directory
prooffix scan .

# Scan specific directory
prooffix scan ./src

# Scan single file
prooffix scan ./src/index.ts

Common Flags

  • --auto-fix - Automatically apply verified fixes
  • --mode=guided-refactor - Interactive refactoring mode
  • --ci - CI-friendly output with SARIF reports
  • --format=[json|sarif|markdown] - Output format
  • --severity=[error|warning|info] - Minimum severity to report
  • --no-snapshot - Skip Git snapshot (use with caution)
  • --config=[path] - Path to config file

prooffix rollback

Undo the last auto-fix operation.

prooffix rollback

prooffix init

Initialize a ProofFix configuration file.

prooffix init

Configuration

Create a .prooffixrc.json or prooffix.config.js in your project root:

{
  "languages": ["typescript", "javascript", "python"],
  "exclude": ["node_modules", "dist", ".git"],
  "rules": {
    "security": "error",
    "performance": "warning",
    "style": "info"
  },
  "autoFix": {
    "enabled": true,
    "requireConfirmation": false
  },
  "snapshot": {
    "enabled": true,
    "branch": "prooffix-backup"
  }
}

CI Integration

ProofFix works seamlessly in CI/CD pipelines.

GitHub Actions

name: ProofFix Scan
on: [push, pull_request]

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm install -g prooffix
      - run: prooffix scan . --ci
      - uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: prooffix-report.sarif

Exit Codes

  • 0 - No issues found
  • 1 - Issues found (warnings or errors)
  • 2 - Critical errors that need immediate attention

Frequently Asked Questions

Does ProofFix send my code to the cloud?

No. ProofFix is 100% local-first. Your code never leaves your machine. There's no telemetry, no cloud processing, no data collection.

What languages are supported?

Currently: TypeScript, JavaScript, Python, Go, Rust, and Java. More languages are being added regularly.

Can I undo changes made by auto-fix?

Yes! ProofFix creates a Git snapshot before making any changes. Run prooffix rollback to undo.

Is ProofFix free?

Yes. ProofFix is open source under the MIT license.

How is this different from other linters?

ProofFix goes beyond linting. It verifies proposed changes, creates rollback points, and ensures every fix is safe before applying.
